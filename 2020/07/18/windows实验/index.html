Windows实验

实验目的，搭建故障转移群集，把web与FTP服务的物理目录存放在故障转移服务器上，断开其中一个故障转移节点，验证能否访问web域ftp。

实验环境

主机名 IP
S4-域控 192.168.128.30/24
S5-节点 192.168.128.31/24 192.168.3.2/24 192.168.4.2/24
S6 -节点 192.168.128.32/24 192.168.3.3/24 192.168.4.3/24
S10-文件 192.168.4.4/24

设置 S4的IP地址。

关机状态为S5、S6添加两张网卡，并设置好各个网卡的IP地址，这里需要我设置的IP比较多，而且虚拟机过多导致我电脑很卡就不逐个截图。

设置S10的IP地址

在S4创建域2020kk.com环境并升级为域控，创建用户user1,user2。A2，A3加入到域中。

为S4安装域服务。

设置S4为域控

根据题目要求设置域名为2020kk.com

域安装成功之后，把S5和S6加入域，并创建两个用户

)))

为了防止防火墙干扰实验，这里关闭所有计算机的防火墙

在S10上创建iscsi服务，并划分磁盘空间；192.168.3.0/24网段为心跳网络；192.168.4.0/24网段为存储网络，S5,S6为节点网络。ISCSI目标命名为A-A。S5，S6,S10组成ISCSI群集。

设置为10G并添加两个节点的IP地址。

然后创建仲裁硬盘

进入S5，S6通过ISCSI发起程序连接S10

S5，S6完成连接之后进入磁盘管理，联机并初始化磁盘

在S5,S6安装文件服务器和故障转移群集

安装完成后进入故障转移群集的控制面板

首先进行验证配置

通过测试。

开始创建群集

创建成功，添加角色。

现在我的群集文件夹在我的节点一上，也就是我的S5上，在S5群集文件夹创建一个共享目录

然后创建一个文件

去节点二查看，也就是我的S6

然后故意断开（强制关机）节点一的连接，也就是我的S5（模拟节点一发生故障）群集服务器会自动转移到我的节点二上，也就是我的S6而且文件夹内容不会丢失。

在server-A2中搭建CA（证书颁发机构）；在server-A1上搭建web以及ftp服务，web1的域名为https://www.2020kk.com，访问时显示hello runtime1111，web2域名为https://eee.2019kk.com,访问时显示hello runtime2222。访问ftp时,用户登录时为自身的家目录，并且在各自的文件夹中创建对应用户名文本。且WEB1和WEB2的默认文档目录存在与ISCSI群集上，网站的IP则是A1的IP.同理，FTP的存储数据文件夹也存在与ISCSI群集上，并且配置完成后做相应的测试.例如:断开ISCSI主用节点时，备份节点继承，web服务与ftp服务不会down。
